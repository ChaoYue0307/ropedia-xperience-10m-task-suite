Xperience-10M 12-task suite, first.

The top-level map shows the Xperience-10M sample modalities and the verified 12-task results. Audio is present in the sample MP4 stream, but the current 8,378-d baseline manifest does not featurize it.

From raw episode to checked artifacts.

Every script works from one data contract: aligned multimodal windows, explicit labels, cached feature extraction, and a manifest that makes omitted modalities visible.

What this project proves

It proves the full engineering loop: reading Xperience-10M sample data, aligning modalities, converting them into model-ready windows, defining meaningful tasks, producing metrics, and packaging every artifact for review.

What it does not prove

It does not claim general embodied intelligence. A single episode cannot support cross-environment generalization; that requires many episodes and held-out episode splits.

Small baselines, no hidden machinery.

Baseline 1: minimal heads

Softmax, logistic, ridge, and retrieval heads keep every input/output contract readable. They are the first sanity check for whether a task is well-posed.

Baseline 2: neural MLP heads

Small PyTorch MLP classifiers/regressors reuse the same features and splits. They test nonlinear gains before heavier Omni fine-tuning.

Junior-friendly task walkthroughs.

Each task is explained as a case study with input, middle process modules, and output. The full generated guide is committed as Markdown and JSON.

timeline_action

Case: a pouring window should map to the current action, such as Pour coffee.

Inputone 20-frame all-modality windowProcesswindow builder, feature assembler, action label builder, classifier, evaluatorOutputcurrent action class

timeline_subtask

Case: a fine action is grouped into a broader drink-preparation stage.

Inputone all-modality windowProcesswindow builder, feature assembler, subtask label builder, classifier, evaluatorOutputcurrent subtask label

transition_detection

Case: detect the moment the demonstrator changes from preparing to pouring.

Inputone window plus boundary labelsProcessboundary builder, feature assembler, binary classifier, boundary matcher, evaluatorOutputboundary or steady

next_action

Case: a preparing-to-pour window should predict the action 20 frames later.

Inputcurrent window at time tProcessfuture label shift, current-only features, classifier, evaluatorOutputaction at t+20 frames

hand_trajectory_forecast

Case: when a hand moves toward a cup, predict the future 3D hand path.

Inputcurrent all-modality windowProcessfuture mocap target builder, regression head, trajectory reshaper, MPJPE evaluatorOutputfuture left/right hand joints

contact_prediction

Case: decide whether the hand/body is touching an object or surface.

Inputnon-contact, non-caption featuresProcessfeature selector, contact target builder, binary classifier, degeneracy checkOutputcontact or no contact

object_relevance

Case: during pouring, infer relevant objects such as milk, cup, or coffee.

Inputnon-caption featuresProcessobject vocabulary, multi-hot targets, sigmoid heads, micro-F1 evaluatorOutputmulti-label object set

caption_grounding

Case: a query like Pour milk into coffee should retrieve the matching moment.

Inputcaption query plus candidate windowsProcessquery builder, candidate builder, projection head, cosine ranker, MRR evaluatorOutputranked matching windows

cross_modal_retrieval

Case: motion and IMU from pouring should retrieve the matching depth/video window.

Inputmotion/IMU/camera query, depth/video candidatesProcessmodality splitter, projection head, candidate index, cosine ranker, top-k evaluatorOutputranked depth/video windows

modality_reconstruction

Case: infer compressed depth/video features from motion, IMU, and camera pose.

Inputmotion/IMU/camera featuresProcesssource-target split, scaling, regression head, inverse scaling, R2 evaluatorOutputdepth/video feature vector

temporal_order

Case: tell whether reaching then pouring has been reversed.

Inputtwo adjacent windowsProcesspair builder, feature combiner, binary classifier, F1 evaluatorOutputcorrect or reversed order

misalignment_detection

Case: detect motion from pouring paired with video/depth shifted later.

Inputmotion side plus visual/depth sideProcessaligned pair builder, shifted pair builder, feature combiner, binary classifierOutputaligned or shifted

Reproduce the suite.

Raw Xperience-10M data is not redistributed here. Download the public sample separately, then run the same scripts.

git clone https://github.com/Ropedia/HOMIE-toolkit.git
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r HOMIE-toolkit/requirements.txt huggingface_hub hf_xet
pip install torch

hf download ropedia-ai/xperience-10m-sample \
  --repo-type dataset \
  --local-dir data/sample/xperience-10m-sample

git clone https://github.com/ChaoYue0307/ropedia-episode-task-suite.git
cd ropedia-episode-task-suite
python scripts/episode_task_suite.py --workspace /path/to/workspace --include-neural
python scripts/generate_visualizations.py
Built as a single-episode embodied-AI learning lab. For real model claims, scale to multiple episodes and hold out entire episodes at evaluation time.
